# SAM Framework — Sparse API Monolith

## What this project is
SAM (Sparse API Monolith) is a developer toolset that lets mid-market engineering teams (Kroger, Capital One, etc.) work in a single API monorepo without downloading the entire codebase. Developers only fetch the domains they own, plus AI-resolved dependencies. The experience should feel like working on a single microservice — fast clone, local editor, targeted tests, safe deploy.

**Mac-first MVP. No Windows until v2.**

## The core developer experience we are building
```
sam init git@company.com/enterprise-api   # 8 sec clone, all folders visible, no code yet
sam use --profile sales-api               # fetches your domain + AI-resolved deps only
# edit code in VS Code — only your domain visible, impact shown inline
git push                                   # MonoWatch pre-push hook shows blast radius
sam deploy --profile sales-api            # deploys only your changed services
```

## Components — what each one does

| Component | Language | Location | Purpose |
|-----------|----------|----------|---------|
| **sam-core** | Rust | `sam-core/` | Core library with C FFI. Git ops, profile parsing, MonoGraph client |
| **sam CLI** | Rust | `sam/` | The `sam` binary. init, use, fetch, plan, impact, deploy, graph commands |
| **MonoGraph** | Python 3.12 | `monograph/` | AI dependency engine. Static import analysis + git co-change mining |
| **MonoLens** | TypeScript | `monolens/` | VS Code extension. Ghost folders, hydrate-on-click, inline impact gutter |
| **MonoWatch** | Go 1.22 | `monowatch/` | Pre-push git hook. Impact analysis before every push |
| **MonoShell** | Swift 5.9 | `monoshell/` | macOS FileProvider extension (iCloud-like). Rust FFI, no FinderSync |
| **(legacy) sam-cli** | Go 1.22 | `sam-cli/` | Original Go CLI — replaced by Rust `sam/` and `sam-core/` |

## Tech stack and conventions

### sam-core (Rust library)
- Rust 1.77+, Cargo workspace member
- Crate types: `lib` + `staticlib` (produces `libsam_core.a` for Swift linking)
- Modules: git, profile, workspace, monograph, output, ffi, error
- C FFI via `#[no_mangle] extern "C"` functions, header generated by cbindgen
- All git ops via `std::process::Command` wrapping the `git` binary
- HTTP client: reqwest (blocking) for MonoGraph daemon
- YAML: serde_yaml for profiles and workspace state

### sam CLI (Rust)
- Rust 1.77+, depends on sam-core
- clap for CLI commands (derive API)
- Binary name: `sam`, installed via Homebrew or `cargo install`
- Commands: init, use, fetch, plan, impact, deploy, graph
- All output to stderr (colored), JSON to stdout (for piping)

### MonoGraph (Python)
- Python 3.12, managed with `uv`
- tree-sitter for multi-language AST parsing (JS, TS, Python, Java, Go, C#)
- Runs as a local HTTP daemon: `monograph serve --port 7474`
- Caches dependency graph to `.sam/graph.json`
- Mines `git log` for co-change frequency scores
- FastAPI for the HTTP interface

### MonoLens (TypeScript / VS Code)
- VS Code Extension API (vscode ^1.85.0)
- TypeScript strict mode
- Reads `.sam/graph.json` and `.sam/profiles.yaml`
- Custom TreeDataProvider for the file explorer sidebar
- Calls `sam` CLI subprocess for hydration
- Extension ID: `sam-framework.monolens`

### MonoWatch (Go)
- Installed as a git pre-push hook by `sam init`
- Reads `git diff --name-only` to get changed files
- Queries MonoGraph daemon for impact analysis
- Outputs colored terminal report; exits non-zero only if `block_on_critical: true` in config

### MonoShell (Swift)
- Xcode 15+, macOS 15.1+ target (Sequoia)
- Uses `FileProvider` framework (NSFileProviderReplicatedExtension) — iCloud-like UX
- NO FinderSync — all visual state managed by FileProvider natively
- Calls sam-core Rust library via C FFI (no subprocess spawning, no XPC)
- Links `libsam_core.a` statically, bridges via `sam_core.h`
- Requires Apple Developer account + entitlements
- App bundle: `SAMShell.app` (menu bar only, no Dock icon)

## Build and run commands

```bash
# sam CLI + sam-core (Rust)
cargo build --release
# Binary at target/release/sam, static lib at target/release/libsam_core.a
# C header at sam-core/target/sam_core.h

# Install sam CLI
cargo install --path sam

# MonoGraph daemon
cd monograph && uv run monograph serve --port 7474

# MonoLens VS Code extension
cd monolens && npm install && npm run compile && code --install-extension .

# MonoWatch (install hook in a test repo)
cd monowatch && go build -o monowatch . && ./monowatch install --repo /path/to/test-repo

# MonoShell (requires Xcode 15+, macOS 15.1+)
# 1. Build sam-core first: cargo build --release
# 2. Open Xcode project:
cd monoshell && open SAMShell/SAMShell.xcodeproj
# 3. Link libsam_core.a and add sam_core.h bridging header in Xcode
# 4. Build and run SAMShell scheme
```

## profiles.yaml schema (the central config)

```yaml
# .sam/profiles.yaml — committed to the repo
profiles:
  sales-api:
    domains:
      - apis/sales
      - apis/pricing
    auto_include:
      - shared/auth      # always hydrated
      - shared/types
    ai_infer: true       # MonoGraph adds deps it finds
    watch:
      - apis/inventory   # alerts only, not hydrated
  products-api:
    domains:
      - apis/products
      - apis/catalog
    auto_include:
      - shared/auth
      - shared/types
      - shared/media
  platform:
    domains: "*"         # full monolith
    ai_infer: false
```

## MonoGraph HTTP API (port 7474)

```
GET  /health                          → {status: "ok"}
POST /analyze        {repo_path}      → triggers full graph build
GET  /graph          ?domain=apis/sales → dependency subgraph
POST /resolve        {domains[]}      → returns all deps to hydrate
POST /impact         {changed_files[]}→ returns affected domains + risk scores
GET  /cochange       ?file=path       → co-change partners with scores
```

## Critical design rules — do not violate

1. **The `sam` CLI must work without the MonoGraph daemon running.** Fall back to static profile resolution if daemon is unreachable. Never block the developer.
2. **MonoGraph never modifies the working tree.** It is read-only. Only `sam fetch` writes files.
3. **profiles.yaml is committed to the repo.** Never gitignore it. It is team-shared config.
4. **`.sam/graph.json` is gitignored.** It is a local cache, regenerated per machine.
5. **MonoWatch pre-push hook is advisory by default.** It warns but never blocks unless `block_on_critical: true` is set in `.sam/config.yaml`.
6. **All sam CLI commands must complete in under 3 seconds** for common operations (use, fetch single domain). Network/git operations are the exception.
7. **MonoShell calls sam-core via FFI.** No subprocess spawning. All git operations go through sam-core Rust library.
8. **MonoShell FileProvider is read-only.** createItem/modifyItem/deleteItem return unsupported. Only hydration (download) and eviction are allowed.
9. **No FinderSync, no polling, no custom context menus.** FileProvider framework handles all Finder UX natively.
