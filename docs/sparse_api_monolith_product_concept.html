How the ghost FS works
The OS shell extension registers a virtual filesystem driver. Every folder in the monorepo appears immediately at clone — but only as a stub inode with metadata. No code is on disk until you open it.
1
git clone --filter=blob:none downloads tree structure only. All folders visible, no file content.
2
Shell extension (Finder/Explorer) marks unhydrated dirs with ghost overlay icon + grayed opacity.
3
Double-tap / enter triggers AI resolver: scans imports + git log co-changes + semantic similarity to fetch exactly what you need.
4
Shared modules (auth, types, utils) hydrate automatically when any domain that needs them is opened — never manually.

How MonoGraph scores dependencies
Static import graph
tree-sitter parses all languages. Builds a directed graph of every import statement across the monolith.
Git co-change frequency
Files changed together in the same commit have high co-change scores. Predicts "you'll probably need this too."
Semantic code embeddings
Embeds all functions. High cosine similarity = likely related even without explicit imports. Catches indirect coupling.
Runtime call traces (optional)
If OpenTelemetry traces are available, actual production call frequency overrides static analysis scores.
MonoFetch CLI — core commands
# Initial setup — tree only, no blobs $ sam init enterprise-api-monolith → clones tree (89 folders, 42 MB) → installs MonoShell extension → all folders appear ghost in Finder # Hydrate by profile $ sam use --profile sales-api → resolving dependencies via MonoGraph → fetching: apis/sales (12 MB) → fetching: shared/auth (3.1 MB) → fetching: shared/types (1.2 MB) → inferred dep: apis/pricing (4 MB) → total: 20.3 MB downloaded # Hydrate a single domain + AI deps $ sam fetch apis/products --with-deps → static imports: shared/types, shared/media → co-change (score 0.87): apis/catalog → semantic sim (score 0.71): apis/search → fetching 4 modules (31 MB) # Check what you'd need before fetching $ sam plan apis/payments → would fetch: 6 modules, 28 MB → critical shared deps: auth, types → impact score: HIGH (touch carefully) # Pre-push impact check $ sam impact → changed: shared/auth/token-validator → affects: 9 domains (2 critical) → blocking push? no (warn only)
